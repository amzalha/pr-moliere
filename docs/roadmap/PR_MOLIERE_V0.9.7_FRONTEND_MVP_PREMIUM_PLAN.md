# PR Molière v0.9.7 — Frontend MVP premium convaincant

## Objectif

Transformer PR Molière v0.9.6 en MVP beaucoup plus convaincant visuellement, sans remplacer le moteur existant.

## Principe central

Ne pas remplacer l’application actuelle.

Conserver :

- React / Vite / TypeScript ;
- Railway ;
- corpus 90 sujets ;
- métadonnées pédagogiques v0.9.6 ;
- tuteur IA existant ;
- vouvoiement global ;
- validation corpus ;
- check anti-tutoiement ;
- build production validé.

Importer seulement les meilleures idées du template :

- accueil plus vendeur ;
- navigation plus claire ;
- présentation MVP plus premium ;
- cartes modules plus attractives ;
- mode démonstration ;
- progression élève plus lisible ;
- badges légers ;
- meilleure expérience mobile ;
- base PWA éventuelle.

## Priorités v0.9.7

### Priorité 1 — Accueil MVP premium

Créer un écran d’accueil plus convaincant avec :

- promesse claire ;
- statistiques fortes ;
- boutons d’action ;
- aperçu des modules ;
- mention du corpus 90 sujets ;
- message adapté enseignants / élèves.

### Priorité 2 — Navigation plus démonstrative

Rendre l’application plus lisible pour une démo :

- sections claires ;
- accès rapide au corpus ;
- accès rapide au tuteur ;
- accès rapide à l’examen ;
- accès rapide à la progression.

### Priorité 3 — Cartes pédagogiques

Améliorer les cartes du corpus :

- difficulté visible ;
- compétence ciblée ;
- critères de réussite ;
- erreurs fréquentes ;
- conseil de progression.

### Priorité 4 — Mobile MVP

Améliorer le rendu téléphone/tablette pour démonstration terrain.

### Priorité 5 — Gamification légère

Ajouter seulement des éléments simples :

- badges ;
- progression ;
- encouragements ;
- niveau de maîtrise.

## Règles strictes

- Aucun tutoiement.
- Ne pas casser v0.9.6.
- Ne pas supprimer le corpus 90 sujets.
- Ne pas intégrer le template brut.
- Ne pas introduire de secrets.
- Ne pas modifier la logique IA avant validation UI.
- Une amélioration à la fois.
- Chaque étape doit passer : corpus, vouvoiement, lint, build.

## Critère de réussite MVP

PR Molière v0.9.7 doit donner une impression de produit sérieux, présentable et testable par :

- un enseignant ;
- un élève ;
- un incubateur ;
- un partenaire EdTech.

## Score cible

- MVP actuel v0.9.6 : 90 %
- MVP cible v0.9.7 : 95 %
